\section{Setting the scene}

\begin{frame}{Swashbuckle – The Good \emoji{thumbsup}
}
Swashbuckle
\begin{itemize}
    \item Popular OpenAPI NuGet package for .NET
    \item \textbf{Shipped by default} in project templates before .NET 9
    \pause
    \item Two responsibilities
    \begin{itemize}
        \item \textbf{Generates OpenAPI specs} (called "Swagger" specs)
        \item Hosts the widely used \textbf{Swagger UI}
    \end{itemize}
    \pause
    \item Supports both Minimal APIs and Controllers \emoji{rocket}

\end{itemize}
\end{frame}


\begin{frame}{Swashbuckle – The Bad \emoji{thumbsdown}}
Swashbuckle
\begin{itemize}
    \item Relies on \textbf{reflection} to inspect \textit{controllers, endpoints, models}
    \item Reads compiled assemblies at \textbf{runtime} to build the OpenAPI spec
    \item Slower startup for large projects; can miss custom attributes
\end{itemize}
\end{frame}

\begin{frame}[fragile]{My Relationship with Swashbuckle \emoji{heart}}
\begin{itemize}
    \item We've had great years together
\end{itemize}
\end{frame}

\begin{frame}[fragile]{My Relationship with Swashbuckle \emoji{thinking-face}}
\begin{itemize}
    \item We've had some great years together
    \item But things changed with .NET 10...
        \begin{itemize}
        \item You can be so complicated...
        \item Native OpenAPI is faster
        \item Documentation just looks better with Scalar
    \end{itemize}
\end{itemize}
\end{frame}

\begin{frame}{OpenAPI Setup .NET 8}
\begin{itemize}
    \item AddEndpointsApiExplorer() → \textbf{Discovers} Minimal API + expose metadata

  \item AddSwaggerGen() → \textbf{Generates} OpenAPI/Swagger document at \runtime using reflection.

  \item UseSwagger() → \textbf{Serves} the generated \textbf{OpenAPI JSON} document over HTTP.

  \item UseSwaggerUI() → \textbf{Serves} the interactive \textbf{Swagger UI{}} to interact with APIs.
\end{itemize}
\end{frame}

\begin{frame}[fragile]{OpenAPI Changes in .NET 10}
\begin{itemize}
    \item \textbf{OpenAPI generator defaults to 3.1}
    \item Key Benefits:
        \begin{itemize}
            \item Full JSON Schema support
            \item Stronger type validation
            \item Clearer required/optional fields
        \end{itemize}
        \pause
    \item Example;
    \begin{itemize}
        \item v3.0; Nullable types required separate "nullable": true property.
        \item v3.1; Fully JSON Schema compliant:
        \begin{codebox}
            "type": ["string", "null"]
        \end{codebox}
    \end{itemize}
\end{itemize}
\end{frame}

\section{Takeaways}
\begin{frame}{Key Takeaways}
\begin{itemize}
  \item Native OpenAPI is lighter and more flexible
  \begin{itemize}
      \item OpenAPI generated at build-time without reflection
      \item Enforce OpenAPI v3.0 for API Management
  \end{itemize}
  \item Transformers allow us to tailor our OpenAPI specifications.
  \begin{itemize}
      \item Only public/private APIs in document
  \end{itemize}
  \item Swagger can still be used; however, I recommend breaking up and using Scalar.
  \begin{itemize}
      \item Modern features
      \item Stays performant
  \end{itemize}
\end{itemize}
\end{frame}

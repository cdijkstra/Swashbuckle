\section{Setting the scene}

\begin{frame}{Swashbuckle – The Good}
Swashbuckle
\begin{itemize}
    \item Popular NuGet package for .NET
    \item \textbf{Shipped by default} in project templates before .NET 9
    \pause
    \item Two responsibilities
    \begin{itemize}
        \item \textbf{Generates OpenAPI specs} (called "Swagger" specs)
        \item Hosts the famous \textbf{Swagger UI}
    \end{itemize}
    \item Works with Minimal APIs and Controllers.
\end{itemize}
\end{frame}


\begin{frame}{Swashbuckle – The Bad}
Swashbuckle
\begin{itemize}
    \item Relies on \textbf{reflection} to inspect \textit{controllers, endpoints, models}
    \item Reads compiled assemblies at \textbf{runtime} to build the OpenAPI spec
    \item Slower startup for large projects; can miss custom attributes
\end{itemize}
\end{frame}

\begin{frame}[fragile]{Swashbuckle – The Status}
Swashbuckle
\begin{itemize}
    \item No longer shipped by default in .NET
    \item \textit{Can} still be included in  your projects
    \pause
    \item But let's break old habits ;) 
\end{itemize}

\begin{codebox}
Farewell Swashbuckle

We're breaking up.
It's not you, it's me... 
Actually, it's .NET 10.
\end{codebox}
\end{frame}

\begin{frame}[fragile]{.NET 9 Changed the Defaults}
    \begin{itemize}
      \item \textbf{OpenAPI Docs} are now generated natively.
    
      \item NuGet
          \begin{verbatim}
              Microsoft.AspNetCore.OpenApi
          \end{verbatim}
        was introduced in the ASP.NET Core shared framework in .NET 9
    \pause
     \item \textbf{OpenAPI UI}: Microsoft intentionally stopped shipping a UI out of the box.
    \end{itemize} 
\end{frame}

\begin{frame}{Minimal OpenAPI Setup (.NET 8)}
\begin{itemize}
    \item AddEndpointsApiExplorer() → Discovers Minimal API endpoints and exposes their metadata for tools like OpenAPI.

  \item AddSwaggerGen() → Generates the OpenAPI/Swagger JSON document at runtime using reflection.

  \item UseSwagger() → Serves the generated OpenAPI JSON document over HTTP.

  \item UseSwaggerUI() → Serves the interactive Swagger UI to browse and test the API.
\end{itemize}
\end{frame}

\begin{frame}[fragile]{Minimal OpenAPI Setup (.NET 10)}
\begin{center}
    \includegraphics[scale=0.2]{Images/net10.png}
\end{center}
\end{frame}

\begin{frame}{Minimal OpenAPI Setup (.NET 10)}
\begin{itemize}
    \item AddOpenApi() → Registers OpenAPI document generation and optional transformers for Minimal APIs or controllers.

  \item MapOpenApi() → Maps an endpoint that serves the generated OpenAPI JSON document at runtime.
\end{itemize}
\end{frame}

\tussenslide{Transformers}


\begin{frame}[fragile]{Transformers}
Nice, we are now generating docs using v3.0 or v3.1. \newline \newline

Three kinds of Transformers can be used within builder.Services.AddOpenApi to tweak documents

\begin{codebox}
    AddSchemaTransformer(),
    AddOperationTransformer()
    AddDocumentTransformer(),
\end{codebox}

\end{frame}

\begin{frame}[fragile]{Document Transformers}
\begin{itemize}
    \item \textbf{AddSchemaTransformer}
    Applies transformations per schema. Adjust data type, apply constraints, add description.
    \pause
    \item \textbf{AddOperationTransformer}
    Applies transformations per operation. Set tags, deprecations, etc. \newline  
    You can also define which endpoints should be included in an OpenAPI document.
    \pause
    \item \textbf{AddDocumentTransformer}
    Applies transformations globally. Set title, description, version, etc.
\end{itemize}
\end{frame}

\begin{frame}[fragile]{Document Transformers}
Short demo
\begin{itemize}
    \item Group endpoints; Publicly available versus Admin
    \item Group endpoints; V1 versus V2
\end{itemize}
\end{frame}

\section{Takeaways}

\begin{frame}{Key Takeaways}
\begin{itemize}
  \item Native OpenAPI is lighter and more flexible
  \begin{itemize}
      \item Enforce Openapi v3.0 for API Management
  \end{itemize}
  \item Document and endpoint transformers allow us to tailor our OpenApi specs
  \begin{itemize}
      \item Only public/private APIs in document
      \item Mark APIs as deprecated
  \end{itemize}
  \item Swagger can still be used; however, I recommend breaking up and using Scalar.
\end{itemize}
\end{frame}

\begin{frame}{Key Takeaways}
\begin{itemize}
  \item OpenAPI generated at build-time without reflection
  \item Transformers allow you to tweak the document
  \item UI renders the experience, various options available
\end{itemize}
\end{frame}
